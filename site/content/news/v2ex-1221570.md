体验了，丝滑。

Intel 低压 U 没插电的笔记本都响应速度飞快。

我还记得以前笔记本升级到 Windows 11 后，很长一段时间都只靠组合键走搜索来快速开应用，因为 Win11 开始菜单布局太低效了。

之后回退 Windows 10 接着用回了宽大的贴一堆磁贴的开始菜单，然后到今天主力机都没升 Win11 。

也就是因为是 Windows 10 ，测试了 op 的 Launcher 看到了好一些被微软坑出的小细节问题，刚好也在这反馈下，
一个是菜单图标字体缺失，这个微软官方 Demo 程序都有撞见过，估计因为用了 Windows 11 才有的资源，在 Win10 上直接变豆腐块；
另一个就是微软删了也没删干净的特色大方块图标，下面图里面可以看到它没有像 Windows 11 一样用透明底，而还是这种和桌面图标以及任务管理器进程图标一样的主题色方块底的 Icon 。

微软 Win10 上是给这种样式做过删改的，但是看样子只改了系统任务栏与开始菜单的显示方式。

(截图是 200% 缩放比的屏幕。

PWA 应用也有方块底色)
另外的细节大约是触摸设备的交互吧，拿触摸屏试了下长按右键和拖拽空白位置翻页完全没问题，不过如果翻页的时候按在图标上，就触发和鼠标拖拽一样的重新排序动作了。

如果要适配触摸估计得特化做个长按手势。

然后，默认情况下可以聚焦搜索框，这样直接 Win + Q 就可以接键盘输入应用名字了。

但是考虑到 Windows 对于没有连接键盘的触屏设备，自动聚焦搞不好会让触摸键盘自动弹出来遮挡屏幕，这种坑我拿不准存不存在、是否值得留个手动设置的开关。

总觉得 Windows 这种硬件排列组合的自由度对开发者来说就是个大难题，微软自己这么多年都没做好。

顺便，小 bug 方面比如搜索框搜索出的图标不能右键，拖拽的图标跨 2 页放下后暂时会不显示，搜索出的文件夹点击展开会变成展开默认排序的同位置文件夹之类，这些可能也都能手动测出来。

点来点去还没遇到什么能崩掉的大的问题。

所以这里主要应该还是 Win10 和 Win11 系统图标差异的这个算是最突出的 (也是被微软坑到的) 小问题。

整体上感受还是极为美丽的，原生应用性能就是好。

感谢 OP 的这个工具，也期待下后继的开发。

我已经想给自己 Win11 的开发机都塞上一份这个 Launcher 了。

刚刚，微软跟进谷歌支持 Go 语言开发 AI 智能体，OpenAI 与 Anthropic 落后一步

⚡ 一句话看懂：刚刚，微软跟进谷歌支持 Go 语言开发 AI 智能体，OpenAI 与 Anthropic 落后一步。来源：InfoQ AI。

Go 语言已成为云基础设施领域的通用标准语言，从容器编排、CI/CD 流水线，到工程师日常离不开的各类命令行工具，无处不在。

Kubernetes、Docker 和 Terraform 都是用 Go 编写的，时至今日，它仍然是团队构建后端服务的首选语言。

与此同时，AI 智能体已成为几乎所有主流软件厂商的焦点，各家都在竞相为开发者提供原生构建智能体的框架。

正因如此，微软现在正将自家智能代理框架引入 Go 语言生态，让云原生开发者能够用日常开发各类业务的 Go 语言来构建 AI 智能体。

最近开始公开预览的微软 Agent Framework for Go 本质上为 Go 语言开发者提供了许多与 Python 和 .NET 开发者相同的构建模块：支持来自 Microsoft Foundry、Azure OpenAI、Anthropic 和 Gemini 的模型；支持工具调用和 MCP，可实现智能体连接外部系统；同时能够调度多个智能体协同完成同一任务。

微软高级软件工程师 Quim Muntal 在发布官方博文中提到，这个智能体框架面向的是那些构建功能更强大、运行周期更长的 AI 系统开发者。

Muntal 写道：“Microsoft Agent Framework 专为那些正从单一提示词调用转向生产级智能体系统的开发者而设计。

这些智能体可调用工具、保持上下文信息、与其他智能体协调、流式传输结果，并且能够作为真实业务应用的一部分接受监控和治理。

” 这正是 Go 语言已经占据的领域：后台服务、命令行工具、后台任务程序和云原生应用，长久以来 Go 都是这类场景的首选开发语言。

本次推出的 Go SDK 将这些能力直接交到在该领域构建的开发者手中。

微软 Agent Framework 适配 Go 语言的历程 简单回顾一下：微软于 2025 年 10 月推出 Agent Framework，作为一个用于开发 AI 智能体与多智能体系统的开源工具包，将此前的两个微软项目 AutoGen 和 Semantic Kernel 统一为一个受支持的平台。

它在今年 4 月份正式全面可用，在 6 月份的 Build 开发者大会上，微软又增加了一系列新功能和工具：用于生产环境（如上下文管理和人工审批步骤）的“智能体 harness”；通过 Microsoft Foundry 提供的托管智能体；一种更快的工具调用方法 CodeAct；以及支持通过新的交接模式将多个智能体串联在一起。

在此之前，Agent Framework 仅支持 .NET 与 Python 两种语言。

社区中有不少开发者都呼吁官方适配 Go 和 Rust。

早在 2025 年 10 月，微软产品经理就曾证实团队正在考虑推出该框架的 Go 版本，但当时表示距离上线“至少还需要数月时间”。

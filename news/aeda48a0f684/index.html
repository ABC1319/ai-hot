深度解读：昆仑万维天工 3.1 来了：上线设计画布与多智能体工作流，强化复杂项目交付能力

⚡ 一句话看懂：深度解读：昆仑万维天工 3.1 来了：上线设计画布与多智能体工作流，强化复杂项目交付能力。来源：InfoQ AI。

6 月 17 日，昆仑万维旗下 Skywork 发布 3.1 版本，面向复杂设计项目和规模化任务处理，上线两项核心能力：Skywork Design 和 Dynamic Workflows。

前者将 AI 设计从传统对话框延展至可视化画布，支持多页面 UI 项目的持续迭代；后者通过多 Agent 并行调度，将复杂任务拆解后分配给多个子 Agent 执行与验证。

官方表示，这次更新是 AI 产品从单次生成向持续交付场景延伸。

过去，用户通过一句 prompt 生成页面、文档或原型，往往可以完成初步展示，但在真实业务场景中，仍需要继续处理多页面一致性、移动端适配、风格统一、版本修改、批量生成和交付格式转换等问题。

对于面向客户展示、内部汇报或开发落地的项目而言，单次生成通常难以覆盖完整需求。

据 Skywork 方面披露，Skywork 3.0 上线后一个月，天工超级智能体收入实现约三倍增长。

在其各类产出能力中，Websites 是用户使用深度较高的场景之一。

部分深度用户并不会在生成一个页面后结束使用，而是围绕同一个项目持续补充需求、调整结构、修改视觉风格和完善页面细节。

Skywork 数据显示，深度用户在单个 project 上的平均交互轮数超过 40 轮，部分复杂项目会横跨多次登录逐步完成。

这意味着，部分用户正在将 AI 工具用于更接近“项目交付”的场景，而不仅是一次性问答或生成 Demo。

对于这类需求，产品需要支持更长上下文、更清晰的可视化管理，以及更稳定的多轮迭代能力。

Skywork Design：将设计过程从对话框扩展到画布 Skywork 3.1 中新增的 Skywork Design，主要解决复杂 UI 设计项目在多轮迭代中的管理问题。

在传统对话式交互中，用户与 AI 的沟通按时间线推进。

当项目进入第十轮、第二十轮甚至更多轮修改后，页面结构、配色方案、组件规范和跳转逻辑容易分散在历史对话中，不利于全局查看和持续调整。

尤其是在一个项目包含多个页面、多个模块和多套状态时，仅依靠对话框管理信息会带来较高的认知成本。

Skywork Design 将设计过程放到一张可延展画布上。

用户提出需求后，Agent 可以先与用户对齐设计方向，再在画布上生成多个页面。

例如，用户输入“做一款健康管理 App，用荧光绿作为主色调”，系统可生成首页、内容生成页、画廊、个人中心等页面，并以画布形式展示完整结构。

在后续修改中，用户可以继续提出“主色调调整为暖色”“按钮尺寸缩小”“新增收藏页面”等需求。

每一轮修改都会对应到画布上的具体页面或组件，用户可以在可视化空间中查看整体变化，而不是依赖历史对话回溯项目状态。

这一设计方式更适合多页面 UI、官网、App 原型、Web App、后台系统等需要长期打磨的项目。

其核心价值不在于单次生成一个页面，而在于让多个页面在持续迭代中保持统一的视觉规范和结构一致性。
